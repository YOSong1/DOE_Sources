# 20.06 프롬프트 엔지니어링 for 실험계획법 분석

역할·맥락·요청 분리·출력 형식의 4요소를 갖춘, 바로 사용할 수 있는 분석용 프롬프트 템플릿 4종입니다.
번호 항목은 검토 관점을 지정하고, 출력 형식은 호출 시 `instructions` 로
JSON 키(`summary`, `risks`, `next_steps`)를 강제합니다.
각 파일은 책 본문의 동일 번호 코드 블록(`# code_20_06_NN.py` 주석)과 1:1 매핑됩니다.

## 파일 구성

| 파일 | 내용 |
|------|------|
| `code_20_06_01.py` | 회귀 분석 결과 해석용 템플릿 |
| `code_20_06_02.py` | ANOVA 결과 해석용 템플릿 (가능/불가능한 결론 구분) |
| `code_20_06_03.py` | DOE 리뷰용 템플릿 (해석 위험·확인 실험 점검) |
| `code_20_06_04.py` | 보고서 Discussion 초안 작성용 템플릿 (4문단 구조) |

## 사용법

```bash
python code_20_06_01.py
python code_20_06_02.py
python code_20_06_03.py
python code_20_06_04.py
```

`OPENAI_API_KEY` 가 없으면 mock 응답으로 흐름이 이어지므로 API 키 없이도 실행할 수 있습니다.
